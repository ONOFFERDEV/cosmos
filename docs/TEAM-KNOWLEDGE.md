# 팀원 개인 지식 가이드 — 내 레포가 곧 내 지식

개인 지식의 정본은 **각자의 GitHub 레포**입니다. 레포에 `.md`를 쓰면(웹에서든 옵시디언에서든) 코스모스가 서버에서 알아서 가져와 **내 개인 공간**에 반영합니다 — PC에 설치할 것이 하나도 없습니다.

내 개인 공간은 본인에게만 보이고(무인증·다른 팀원에게 0건 노출 — 격리 게이트로 검증됨), 공유하고 싶은 것만 골라 공용으로 승격할 수 있습니다.

## 시작 (제일 쉬운 길: 내 AI에게 시키기)

쓰는 AI(클로드 코드·커서 등)에게 이 한 줄을 주면 레포 생성부터 연결·검증까지 알아서 합니다:
> **<코스모스 주소>/web/kit/AI-SETUP.md 읽고 내 지식 레포 세팅해줘** (+ 코스모스 토큰 전달)

## 시작 (수동)

1. **레포 만들기** — [knowledge-template](https://github.com/ONOFFERDEV/knowledge-template)에서 "Use this template"로 내 프라이빗 레포 생성(org 권한이 있으면 `ONOFFERDEV/knowledge-<이름>` 권장 — 토큰 불요).
2. **연결** — 코스모스(관리자가 알려준 주소) 화면 아래 Cosmos 바의 **[📝 내 지식 연결]** 버튼 → 레포 주소(`ONOFFERDEV/knowledge-철수` 형식) 입력 → 연결. 그 자리에서 1회 동기화되며 상태가 표시됩니다.
   - 회사 org 레포는 토큰 입력이 필요 없습니다(서버가 공용 읽기 토큰 사용). 개인 계정 프라이빗 레포만 read 전용 토큰을 함께 입력.
3. **노트 쓰기** — 레포에 자유롭게 `.md`를 만듭니다(GitHub 웹 편집이 제일 간단, 옵시디언/VSCode 클론도 좋음). 파일 하나=지식 하나, 첫 줄 `# 제목`이 문서 제목. push하면 **1시간 내 자동 반영**, 급하면 패널의 [지금 동기화].
4. **확인** — 코스모스 3D 뷰에 **"개인 · " 라벨+점선 링**이 붙은 내 성운이 생기고, 챗 질문에 내 노트가 출처로 인용됩니다.

## 알아둘 것

- **격리**: 내 노트는 검색·3D·다이제스트 어디서도 다른 사람에게 안 보입니다. 폴더 경로·환경 정보 같은 로컬 얘기를 마음껏 적어도 됩니다.
- **레포가 백업**: 지식이 git 이력으로 남으니 버전관리·복구가 공짜입니다.
- `.`으로 시작하는 폴더(.github 등)와 .md 아닌 파일은 수집되지 않습니다. 파일을 지워도 코스모스에서 자동 삭제되지는 않습니다(v1 한계 — 필요하면 관리자에게).
- **공용으로 승격**: 팀 전체가 봐야 할 지식이 생기면 관리자에게 요청 — 지식 PR(브랜치)로 검토→병합되며, 롤백하면 다시 개인으로 돌아옵니다(무손실).

## 문제 해결

| 증상 | 원인/해결 |
|---|---|
| 연결 시 "GitHub 404" | 레포 주소 오타 또는 접근 권한 없음 — org 레포인지 확인, 개인 프라이빗이면 토큰 입력 |
| push했는데 반영 안 됨 | 1시간 주기 대기 중 — 패널 [지금 동기화]로 즉시 반영 |
| 웹에서 내 성운이 안 보임 | 브라우저에 토큰 미저장 — 초대 링크로 한 번 접속(자동 저장) 후 새로고침 |

## (관리자용) 팀원 레포 만들어주기

```powershell
gh repo create ONOFFERDEV/knowledge-<이름> --template ONOFFERDEV/knowledge-template --private
gh api repos/ONOFFERDEV/knowledge-<이름>/collaborators/<깃헙아이디> -X PUT -f permission=push
```
서버 공용 읽기 토큰은 Rocky `deploy/.env`의 `GITHUB_KNOWLEDGE_TOKEN`(fine-grained PAT, ONOFFERDEV org contents:read).
